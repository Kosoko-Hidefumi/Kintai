# 受入・後期が表示されない問題 - 調査結果

## 概要
リポジトリ（デプロイ版）では受入・後期の研修医が表示されないが、ローカルでは表示される。その原因と対策。

---

## 原因の特定

### 1. データの流れ

| 環境 | 研修医一覧のデータソース |
|------|------------------------|
| **ローカル (kintai)** | `resident_dashboard/` に埋め込まれた React アプリ。ユーザーが Excel をアップロードして表示 |
| **リポジトリ (研修医一覧)** | 同じくユーザーが Excel をアップロード。Vercel 等にデプロイされた単体アプリ |

※ 期別リスト（kibetu_list）の `kibetu_result` は研修医一覧とは別。研修医一覧は **独自のファイルアップロード** でデータを読む。

---

### 2. グループ表示のロジック

研修医一覧は **学年（PGY1〜PGY6）** でグループ化している。

- `useDataFilter.ts` の `gradeOrder = ["PGY1", "PGY2", ..., "PGY6"]`
- 学年が PGY1〜PGY6 以外の人は **「その他」** に集約
- `MasterTab` の `gradeOrder` に `"その他"` を含めることで表示

**想定される問題**:
- マスタで「受入」「後期」の人が **学年列が空** または **学年に "受入"/"後期" が入っている** 場合
- 「その他」には入るが、**学年ドロップダウンに「受入」「後期」が無い**ため、フィルタで「受入」や「後期」を選べない
- ただし「すべての学年」なら表示される想定

---

### 3. 列マッピングの違い（重要）

| 処理 | 初・後列の検出方法 |
|------|-------------------|
| **process_data.py / kibetu_list.py** | 列名に「初」「後」「期」のいずれかを含む列を検出 |
| **useExcelParser.ts（研修医一覧）** | **「初・後」という列名のみ** を参照 |

Excel の列名が「期」「初後」など別名の場合、研修医一覧では **初・後列が空** になり、データが正しく読み込まれない可能性がある。

---

### 4. 期別リスト側の除外ロジック（参考）

`process_data.py` と `kibetu_list.py` では次の除外がある:

```python
# 初・後列が「受入」「後期」の行はカウントから除外
exclude_mask = ki_values.str.contains('受入', na=False) | ki_values.str.contains('後期', na=False)
df_filtered = df_master[...] & ~exclude_mask
```

これは **期別リストの集計** 用であり、**研修医一覧の React アプリには直接影響しない**。  
研修医一覧はアップロードされた Excel をそのまま解析しており、この除外は適用されていない。

---

## ローカルとリポジトリの違い

| 観点 | ローカル (kintai) | リポジトリ (研修医一覧) |
|------|-------------------|--------------------------|
| **ビルド** | `kintai/resident_dashboard/` のビルド済みファイルを使用 | 研修医一覧リポジトリのソースをビルドしてデプロイ |
| **ビルド時期** | 過去に別ソースからコピーした可能性 | 最新の main / list-residents 等からビルド |
| **初・後列** | 同様に「初・後」固定でマッピング | 同様 |

**結論**:  
- 研修医一覧の React ソース自体には「受入」「後期」を **除外する処理はない**
- 差は **ビルドの新旧** や **列名のずれ** による可能性が高い

---

## 修正案

### 案1: 初・後列の代替列名に対応（useExcelParser.ts）

`process_data` と同様に、列名に「初」「後」「期」を含む列を検出して使用する。

```typescript
// 初・後列の検出（「初・後」以外の列名にも対応）
function findKiColumn(rows: Record<string, unknown>[]): string | null {
  if (rows.length === 0) return null;
  const first = rows[0];
  for (const key of Object.keys(first || {})) {
    if (key === "初・後" || key.includes("初") || key.includes("後") || key.includes("期")) {
      return key;
    }
  }
  return null;
}
```

### 案2: 「受入」「後期」を独立セクションとして表示（useDataFilter.ts, MasterTab.tsx）

- `gradeOrder` に `"受入"`, `"後期"` を追加
- グループ化時に `初・後 === "受入"` なら「受入」、`初・後 === "後期"`（完全一致）なら「後期」、それ以外は従来通り `学年` でグループ化

```typescript
// useDataFilter.ts の sortedByGrade 内
const gradeOrder = ["PGY1", "PGY2", "PGY3", "PGY4", "PGY5", "PGY6", "受入", "後期", "その他"];

filteredData.forEach((r) => {
  const ki = (r["初・後"] || "").toString().trim();
  let grade = r.学年;
  if (ki === "受入") grade = "受入";
  else if (ki === "後期") grade = "後期";  // "50期後期" は除外（完全一致のみ）
  // ...
});
```

### 案3: 学年ドロップダウンに「受入」「後期」を追加（useDataFilter.ts）

`grades` の算出時に、データ内に「受入」「後期」があればドロップダウンに追加する。

---

## 推奨対応

1. **案1** を実装し、列名違いによる読み込み漏れを防ぐ  
2. **案2** を実装し、「受入」「後期」を独立セクションとして表示  
3. **案3** を実装し、学年フィルタで「受入」「後期」を選択可能にする  

これらを 研修医一覧 の `resident-dashboard` ソースに反映し、ビルドして kintai の `resident_dashboard` にコピーすれば、ローカル・リポジトリともに同じ挙動になる。

## 実装用パッチ（コピー＆適用）

### useExcelParser.ts の修正

`parseFile` 内で、行をパースする前に「初・後」相当の列名を検出する:

```typescript
// parseFile 内、rows の前で
function findKiColumn(rows: Record<string, unknown>[]): string | null {
  if (rows.length === 0) return null;
  const first = rows[0];
  for (const key of Object.keys(first || {})) {
    if (key === "初・後" || (key.includes("初") && key.includes("後")) || key.includes("期")) {
      return key;
    }
  }
  return null;
}

// 使用箇所
const kiCol = findKiColumn(rows) || "初・後";
const parsed = rows.filter((r) => r["名前"]).map((r) => ({
  ...
  "初・後": String(r[kiCol] ?? r["初・後"] ?? ""),
  ...
}));
```

### useDataFilter.ts の修正

1. **sortedByGrade** の grouping を変更:

```typescript
const gradeOrder = ["PGY1", "PGY2", "PGY3", "PGY4", "PGY5", "PGY6", "受入", "後期", "その他"];
// ...
filteredData.forEach((r) => {

  let grade = r.学年 || "";
  if (ki === "受入") grade = "受入";
  else if (ki === "後期") grade = "後期";  // "50期後期"は含めない（完全一致のみ）
  else if (!grade) grade = "その他";

  if (grouped[grade]) {
    grouped[grade].push(r);
  } else {

    grouped["その他"].push(r);
  }
});
```

2. **grades** に受入・後期を追加:

