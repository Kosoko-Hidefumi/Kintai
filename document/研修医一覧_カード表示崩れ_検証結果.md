## 1. 現象

- **変更前**（3eed0bd）: カード型が正常に表示される
- **変更後**（PersonModal + modal_photo_css 適用後）: カード型が表示されなくなる
- **現在**: `git reset --hard` で 3eed0bd に戻し、カードは再び表示

| ファイル | 変更内容 |
|----------|----------|
| **PersonModal.tsx** | 写真レイアウトを `flex-col` に変更。img に `resident-modal-photo` クラスを追加 |
| **app.py** | `modal_photo_css` を `</head>` の直前に注入 |

## 3. 想定される原因（候補）

### 3.1 ビルド出力の差分
- 再ビルドすると Vite が新しいハッシュのファイルを生成（例: index-Cg13M9Sm.js）
- index.html の script/link 参照も変わる
- app.py は `assets/` 内の全ファイルをループし、該当するタグをインライン化する
- **仮説**: 新しい index.html の参照先と app.py の置換が正しく対応していれば問題ない

### 3.2 app.py の置換ロジック
```python
for filename in os.listdir(assets_dir):
    if filename.endswith('.js'):
        html_content = html_content.replace(
            f'<script type="module" crossorigin src="/assets/{filename}"></script>',
            f'<script type="module">{js_content}</script>'
        )
```
- index.html に書かれている `filename` と一致する script タグが 1 つだけ存在する前提
- assets/ に新旧の .js が混在していても、index.html が参照するファイルだけが置換される想定

### 3.3 modal_photo_css 注入
```python
html_content = html_content.replace("</head>", modal_photo_css + "\n</head>", 1)
```
- `</head>` を一度だけ置換
- **懸念**: 注入する CSS が HTML パースや後続の置換に悪影響を与えていないか
- `min()`, `!important` などは一般的なブラウザで問題ない想定

### 3.4 PersonModal の変更
- `flex-col`, `resident-modal-photo`, `max-w-sm` などのクラス追加
- **懸念**: 不適切な JSX やランタイムエラーでアプリ全体が落ちている可能性
- ただし PersonModal はモーダル表示時のみレンダリングされ、初回表示時点ではマウントされない

### 3.5 Streamlit の iframe と HTML サイズ
- ビルド方法を変えていないため、バンドルサイズはほぼ同じ（約 6.4MB）
- 作業時は同一ビルドフロー（`npm run build`）を使用
- **仮説**: サイズが原因である可能性は低い

## 4. 検証手順（段階的テスト）

### テスト A: app.py の CSS 注入のみ
- PersonModal は変更しない（再ビルドなし）
- app.py に `modal_photo_css` 注入だけを追加
- **結果**: カードが**表示されなくなった** → CSS 注入が原因

### テスト B: PersonModal の変更のみ
- app.py は変更しない
- PersonModal だけ変更して再ビルド・反映
- **結果**: カードが表示されれば、原因は CSS 注入側

### テスト C: 両方の変更
- 両方を適用した状態で動作を確認
- **結果**: どちらか片方で現象が出ていれば、その側を詳細に調査

---

## 5. 推奨アプローチ（カードを崩さずにモーダルで写真を大きくする）

### 案 1: CSS のみで対応（ビルド不要）
- app.py で `modal_photo_css` を注入するだけ
- PersonModal の img に既存の `w-20 h-20` のまま、CSS で `.fixed.inset-0 img[src^="data:image"]` をターゲットにしてサイズを上書き
- React 側の変更・再ビルドは一切なし
- リスクが最小

### 案 2: ビルド済み HTML を直接編集
- resident_dashboard/index.html の `<head>` 内に CSS を直接追記
- app.py は変更せず、既存の置換ロジックのまま
- ただし以降の `npm run build` で上書きされうるため、ビルド手順を変える必要あり

### 案 3: 注入用 CSS を外部ファイル化
- `resident_dashboard/modal-photo.css` を作成
- app.py の置換ループで、このファイルも `<style>` として読み込み・注入
- PersonModal は変更しない

---

## 6. 結論

- **優先**: 案 1（app.py の CSS 注入のみ、PersonModal は変更しない）でテストし、カード表示が維持されるか確認する。
- 問題が出なければ、次に PersonModal の変更を段階的に追加して、再現条件を絞り込む。
