# Streamlitアプリの起動方法

## ローカルでの動作確認（必須手順）

**ローカルで動作確認するときは、必ず次の手順で起動してください。**

`streamlit run app.py` だけで起動しないでください。PC 全体に入っている別の Python が使われ、Google Sheets 接続で SSL エラーになることがあります（Streamlit Cloud では発生しません）。

### 推奨: 起動スクリプトを使う

```powershell
cd D:\code4biz\Kintai-clone
.\start-local.ps1
```

`start-local.ps1` は次を自動で行います。

1. プロジェクトの `venv` を使う
2. `requirements.txt` から依存関係をインストール
3. **Windows ローカル専用**の SSL パッケージ（`pip-system-certs` / `truststore`）を追加インストール
4. `.\venv\Scripts\python.exe -m streamlit run app.py` で起動

> SSL パッケージは **Cloud 用の `requirements.txt` には含めない**。Cloud に入れると起動が止まることがある（詳細: `document/Streamlit_Cloud_ロード停止_障害対応記録.md`）。

起動後、ブラウザで **http://localhost:8501** を開いて確認してください。

### 手動で同等の手順を実行する場合

スクリプトを使わない場合は、以下と **同じ内容** を実行してください。

```powershell
cd D:\code4biz\Kintai-clone
.\venv\Scripts\python.exe -m pip install -r requirements.txt
.\venv\Scripts\python.exe -m streamlit run app.py
```

> **注意:** `venv\Scripts\activate` のあと `streamlit run app.py` でも動く場合がありますが、PATH の状態によってはグローバル Python が使われることがあります。確実性のため `.\start-local.ps1` または上記の `python.exe -m streamlit` 形式を使ってください。

## 初回セットアップ

初めてクローンした場合や `venv` がない場合:

```powershell
cd D:\code4biz\Kintai-clone
python -m venv venv
.\start-local.ps1
```

`.streamlit/secrets.toml` に Google 認証情報が未設定の場合は、README または `Google_Sheets設定手順.md` を参照してください。

## なぜ Streamlit Cloud とローカルで挙動が違うのか

| 項目 | Streamlit Cloud | ローカル（Windows） |
|------|-----------------|---------------------|
| OS | Linux | Windows |
| Python | クラウド環境固定 | PC に複数入っていることがある |
| SSL 証明書 | 問題になりにくい | システム証明書の読み込みが必要 |
| 認証情報 | Streamlit Secrets | `.streamlit/secrets.toml` |

コードやスプレッドシート設定が正しくても、**起動に使う Python が違う**とローカルだけ失敗することがあります。

## トラブルシューティング

### エラー: スプレッドシートの取得に失敗 / PermissionError(13)

**よくある原因:** `streamlit run app.py` を venv 外で実行した

**対処法:**

1. 起動中の Streamlit を `Ctrl+C` で停止
2. `.\start-local.ps1` で再起動
3. ブラウザを強制リロード（`Ctrl + Shift + R`）

それでも失敗する場合は `.streamlit/secrets.toml` の設定を確認してください。

### エラー: "streamlit が見つかりません"

**原因:** 仮想環境の Python を使っていない

**解決方法:**

```powershell
.\venv\Scripts\python.exe -m pip install -r requirements.txt
.\start-local.ps1
```

### エラー: "実行ポリシーによりスクリプトの実行が無効になっています"

**原因:** PowerShell の実行ポリシーが制限されている

**解決方法:**

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

その後、再度 `.\start-local.ps1` を実行

### ポートが既に使用されている

**エラーメッセージ:** "Port 8501 is already in use"

**解決方法:**

1. 既存の Streamlit を `Ctrl+C` で停止
2. または別ポートで起動:
   ```powershell
   .\venv\Scripts\python.exe -m streamlit run app.py --server.port 8502
   ```

### Streamlit Cloud: ロード画面のスピナーが終わらない

**よくある原因:**

- `requirements.txt` に Windows 専用 SSL パッケージが入っている
- `app.py` で `importlib.reload` している
- デプロイ直後で再ビルドが未完了

**対処法:**

1. [share.streamlit.io](https://share.streamlit.io) → 対象アプリの **「︙」→ Reboot**
2. 2〜3 分待って URL を強制リロード（`Ctrl + Shift + R`）
3. ログ: **アプリ URL 右下「Manage app」→ Logs**（ダッシュボードの「︙」には Logs はない）

**詳細な障害記録:** `document/Streamlit_Cloud_ロード停止_障害対応記録.md`

## 起動成功の確認

起動が成功すると、以下のようなメッセージが表示されます:

```
You can now view your Streamlit app in your browser.

Local URL: http://localhost:8501
```

ブラウザが自動的に開かない場合は、表示された URL をブラウザで開いてください。
