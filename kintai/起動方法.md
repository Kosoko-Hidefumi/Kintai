# Streamlitアプリの起動方法

## 問題の原因

`streamlit run app.py`が起動しない場合、仮想環境が正しく有効化されていない可能性があります。

## 解決方法

### 1. 仮想環境を有効化する

**PowerShellの場合：**
```powershell
.\venv\Scripts\Activate.ps1
```

**コマンドプロンプトの場合：**
```cmd
venv\Scripts\activate.bat
```

### 2. 仮想環境が有効化されているか確認

プロンプトの先頭に`(venv)`が表示されていることを確認してください：
```
(venv) PS D:\code4biz\kintai>
```

### 3. Streamlitアプリを起動

```powershell
streamlit run app.py
```

## 完全な手順（初回起動時）

```powershell
# 1. プロジェクトディレクトリに移動
cd D:\code4biz\kintai

# 2. 仮想環境を有効化
.\venv\Scripts\Activate.ps1

# 3. 必要なパッケージがインストールされているか確認（必要に応じて）
python -m pip install -r requirements.txt

# 4. Streamlitアプリを起動
streamlit run app.py
```

## トラブルシューティング

### エラー: "streamlit が見つかりません"

**原因：** 仮想環境が有効化されていない

**解決方法：**
1. `.\venv\Scripts\Activate.ps1`を実行して仮想環境を有効化
2. `python -m pip list | findstr streamlit`でstreamlitがインストールされているか確認
3. インストールされていない場合は`python -m pip install -r requirements.txt`を実行

### エラー: "実行ポリシーによりスクリプトの実行が無効になっています"

**原因：** PowerShellの実行ポリシーが制限されている

**解決方法：**
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

その後、再度`.\venv\Scripts\Activate.ps1`を実行

### ポートが既に使用されている

**エラーメッセージ：** "Port 8501 is already in use"

**解決方法：**
1. 別のポートで起動：
   ```powershell
   streamlit run app.py --server.port 8502
   ```
2. または、既存のStreamlitプロセスを終了：
   - タスクマネージャーで`streamlit`プロセスを終了
   - または`Ctrl+C`で既存のプロセスを停止

## 確認事項

起動が成功すると、以下のようなメッセージが表示されます：

```
You can now view your Streamlit app in your browser.

Local URL: http://localhost:8501
Network URL: http://192.168.x.x:8501
```

ブラウザが自動的に開かない場合は、表示されたURLをブラウザで開いてください。
