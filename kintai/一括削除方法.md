# テストデータの一括削除方法

## 方法1: Googleスプレッドシートから直接削除（最も簡単）

### 手順

1. **Googleスプレッドシートを開く**
   - ブラウザでGoogleスプレッドシートにアクセス
   - スプレッドシートIDは、アプリのサイドバー（管理者設定）で確認できます

2. **attendance_logsシートを開く**
   - 下部のタブから「attendance_logs」を選択

3. **データを選択して削除**
   - ヘッダー行（1行目）以外のデータ行をすべて選択
   - 右クリック → 「行を削除」
   - または、行番号をドラッグして選択 → Deleteキー

4. **eventsシートも同様に削除**
   - 「events」タブを選択
   - ヘッダー行以外のデータを削除

### 注意事項

- **ヘッダー行（1行目）は削除しないでください**
- 削除後、アプリをリロード（F5）すると反映されます

## 方法2: 管理者機能として一括削除機能を追加

管理者画面に一括削除機能を追加することもできます。

### 実装例

```python
def delete_all_attendance_logs(spreadsheet_id: str):
    """勤怠ログをすべて削除（ヘッダー以外）"""
    worksheet = get_worksheet(spreadsheet_id, "attendance_logs")
    if worksheet is None:
        return False
    
    try:
        # 全データを取得
        all_values = worksheet.get_all_values()
        if len(all_values) <= 1:  # ヘッダーのみ
            return True
        
        # ヘッダー以外の行を削除
        worksheet.delete_rows(2, len(all_values))
        return True
    except Exception as e:
        st.error(f"削除に失敗しました: {e}")
        return False
```

## 方法3: 特定の条件で削除

### 日付範囲で削除

Googleスプレッドシートで：
1. フィルタを有効化
2. 日付列でフィルタを設定
3. 該当する行を選択して削除

### 職員名で削除

1. フィルタを有効化
2. 職員名列でフィルタを設定
3. テスト用の職員名を選択
4. 該当する行を選択して削除

## 方法4: スプレッドシートを新規作成

完全にクリーンな状態にしたい場合：

1. 新しいスプレッドシートを作成
2. 必要なシート（attendance_logs、bulletin_board、events）を作成
3. ヘッダー行を設定
4. アプリのスプレッドシートIDを新しいものに変更

## 推奨方法

**方法1（Googleスプレッドシートから直接削除）**が最も簡単で安全です。

### 具体的な手順

1. Googleスプレッドシートを開く
2. 「attendance_logs」シートで：
   - 2行目から最後までを選択（Shift + クリック）
   - 右クリック → 「行を削除」
3. 「events」シートで：
   - 2行目から最後までを選択
   - 右クリック → 「行を削除」
4. アプリをリロード（F5）

これで、テストデータがすべて削除され、ヘッダー行のみが残ります。
